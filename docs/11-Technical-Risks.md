# Risks and Technical Debts

| \#  | Risk / debt                                                                                                                                                                                                                                                                                                                            | Mitigation                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
|-----|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| RT1 | **Linux execution surface (resolved).** A devcontainer holospace needs a Linux/POSIX execution surface on a Wasm-only, no-WASI substrate. The choice was a hologram-native CPU/system *emulator-as-container* (arbitrary OCI images) vs. a *Wasm-native* surface of κ-addressed Wasm-recompiled userlands over the substrate host ABI. | **Resolved as ADR-008:** the Wasm-native surface, because the emulator alternative reintroduces location-as-identity (Law L1) and a second execution medium (Law L4). Realized by the `surface` building block and witnessed by `CC-6` (a recompiled userland validates against the WebAssembly spec + host-ABI surface and boots, suspends, resumes, and migrates on the real substrate runtime). Producing a recompiled userland (the libc/toolchain) and multi-module linking remain upstream/future work. |
| RT2 | **Browser `.holo` engine (realized).** Running the [hologram](https://github.com/Hologram-Technologies/hologram) executor in the browser (compiled to Wasm via `wasm-pack`) was the keystone build.                                                                                                                                    | **Realized:** the executor runs in headless Chromium and produces a κ identical to the native executor for the same `.holo` — witnessed live by `scripts/browser-manager-test.sh` (the CI `browser` job; `CC-2`). In-browser **container** boot (a browser `ContainerEngine`) remains future work.                                                                                                                                                                                                            |
| RT3 | **Documentation toolchain weight.** The doc pipeline requires JDK 21, Docker, Ruby, a large Structurizr download, and a built cmark-gfm; it cannot run in a base container without provisioning.                                                                                                                                       | Pinned in the devcontainer and `docs/scripts/install-tools.sh`; exercised in CI (`.github/workflows/ci.yml`). The JDK is pinned to 21 to match the pipeline.                                                                                                                                                                                                                                                                                                                                                  |
| RT4 | **Cross-peer rendezvous.** Discovering an operator’s other instances without a server depends on hologram’s content-addressed networking.                                                                                                                                                                                              | Use hologram’s mechanism (KappaSync) as-is; do not introduce a signaling server or any host/peer-id naming (Law L1).                                                                                                                                                                                                                                                                                                                                                                                          |

This chapter is maintained as risks are resolved (becoming decisions in
Chapter 9) or retired.
