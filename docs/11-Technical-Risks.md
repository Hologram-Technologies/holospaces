# Risks and Technical Debts

| \#  | Risk / debt                                                                                                                                                                                                                                                                                                                            | Mitigation                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
|-----|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| RT1 | **Linux execution surface (resolved).** A devcontainer holospace needs a Linux/POSIX execution surface on a Wasm-only, no-WASI substrate. The choice was a hologram-native CPU/system *emulator-as-container* (arbitrary OCI images) vs. a *Wasm-native* surface of κ-addressed Wasm-recompiled userlands over the substrate host ABI. | **Resolved as ADR-008:** the Wasm-native surface, because the emulator alternative reintroduces location-as-identity (Law L1) and a second execution medium (Law L4). Realized by the `surface` building block and witnessed by `CC-6`: a recompiled userland validates against the WebAssembly spec + host-ABI surface and boots, suspends, resumes, and migrates on the real substrate runtime — on both the native Wasmtime engine and the `wasmi` interpreter engine (the browser + bare-metal `ContainerEngine`). A devcontainer holospace binds and boots the userland its config selects. |
| RT2 | **Browser execution (realized).** Running the [hologram](https://github.com/Hologram-Technologies/hologram) executor and a Wasm `ContainerEngine` in the browser (compiled to Wasm via `wasm-pack`) was the keystone build.                                                                                                            | **Realized:** the `.holo` executor runs in headless Chromium and produces a κ identical to the native executor for the same `.holo` (`CC-2`), **and** the browser peer boots userland containers in-browser via hologram’s `wasmi` interpreter `ContainerEngine` (`CC-6`) — both witnessed live by `scripts/browser-manager-test.sh` (the CI `browser` job).                                                                                                                                                                                                                                     |
| RT3 | **Documentation toolchain weight.** The doc pipeline requires JDK 21, Docker, Ruby, a large Structurizr download, and a built cmark-gfm; it cannot run in a base container without provisioning.                                                                                                                                       | Pinned in the devcontainer and `docs/scripts/install-tools.sh`; exercised in CI (`.github/workflows/ci.yml`). The JDK is pinned to 21 to match the pipeline.                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| RT4 | **Cross-peer rendezvous.** Discovering an operator’s other instances without a server depends on hologram’s content-addressed networking.                                                                                                                                                                                              | Use hologram’s mechanism (KappaSync) as-is; do not introduce a signaling server or any host/peer-id naming (Law L1).                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |

This chapter is maintained as risks are resolved (becoming decisions in
Chapter 9) or retired.
