== Solution Strategy

The strategy follows directly from the laws (Chapter 2) and the conceptual model (Chapter 8).

[options="header"]
|===
| Goal / constraint | Strategy
| Run anything, uniformly (R1–R4) | Treat every workload as a *holospace* — a bootable, κ-addressed unit — and manage all holospaces identically through one lifecycle (boot/suspend/resume/migrate/terminate), regardless of how they were provisioned.
| Two provisioning paths (R1, R2) | A holospace is provisioned either from a *holo-file* (a `.holo` compute artifact, run via the https://github.com/Hologram-Technologies/hologram[hologram] executor) or from a *devcontainer* (a repository + `devcontainer.json`, whose operating system is computed by a κ-addressed system-emulator codemodule over the substrate host ABI — ADR-009; the OS image and repo are κ-addressed content, not a located image). Both yield a κ-addressed holospace definition.
| Everything through the substrate (L4) | holospaces is a thin layer of operations over hologram's storage / network / runtime; it adds no parallel store, cache, or transport. The runner is a `.holo` engine behind hologram's ContainerRuntime; all state lives in the store as κ.
| Bound memory by content addressing (L3) | Hold κ-labels, not objects; resolve content on demand (a κ-resolve is the "page fault"); identical content is stored once (dedup). A holospace far larger than RAM remains bootable.
| Run in every environment (R4) | Provide one environment-agnostic boot core plus a per-environment _peer_ realization: *browser* (the executor compiled to Wasm via `wasm-pack`), *native*, and *bare-metal*. The same holospace κ boots on any peer.
| No server, identity-for-sync (L1, R5) | Sign-in unlocks a self-sovereign key — not a server account. The identity links an operator's instances so their holospaces synchronise over the substrate; suspend on one instance (→ κ) and resume on another.
| Single source of truth | This documentation (arc42 + C4 / OPM / ISO 15288) is authoritative; it precedes code, and every implemented feature links back to it.
|===
