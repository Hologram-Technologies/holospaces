CC-37 — arm64 devcontainer authority artifacts (a stock linux-arm64 binary runs).

The witness (crates/holospaces/tests/cc37_aarch64.rs) assembles the rootfs below
into an ext4 image with the in-crate Layer Assembler (CC-7 / κ-disk content) and
boots the arm64 Linux kernel below on the emulator — which mounts that rootfs
over its virtio-blk device (the shared substrate-backed device bus,
emulator::devbus, GIC-routed) and runs the **stock, unmodified linux-arm64
busybox** as PID 1. The differential oracle is qemu-system-aarch64 -M virt on the
same kernel + rootfs.

linux/Image.gz
  A real, unmodified Linux 6.6.0 arm64 kernel (torvalds/linux v6.6), built for
  this witness like the CC-36 kernel but with **no embedded initramfs**
  (INITRAMFS_SOURCE="") so it mounts a real virtio-blk root, plus VIRTIO_MMIO +
  VIRTIO_BLK + EXT4 + virtio-net/9p + IP autoconf. Reproducible build env
  (KBUILD_BUILD_TIMESTAMP="Fri 31 May 2026 00:00:00 UTC", USER=holospaces,
  HOST=cc37, .version=0); the resolved kernel/.config is committed.
  Toolchain: aarch64-linux-gnu-gcc (Ubuntu 13.3.0).

rootfs/busybox, rootfs/layer.tar.gz
  A **static linux-arm64 busybox 1.36.1** — the canonical glibc stock binary
  (the same busybox the CC-22 riscv64 devcontainer uses, here built for arm64
  with `make ARCH=arm64 CROSS_COMPILE=aarch64-linux-gnu- defconfig` +
  CONFIG_STATIC=y, stripped). layer.tar.gz is a reproducible tar+gzip of
  /bin/busybox, the OCI image layer the Layer Assembler overlays into the rootfs.
  This is the real binary the witness runs: it executes its baseline NEON
  memcpy/memset/strlen (Advanced SIMD is mandatory on AArch64), forks children
  that fault copy-on-write pages, and dispatches applets over fork+exec — all on
  the emulator, with no freestanding shim and no riscv64 workaround.

init.sh
  The arm64 devcontainer's PID 1: the busybox shell brings up the
  pseudo-filesystems, installs the applet symlinks, then runs **unmodified
  linux-arm64 applets** over fork+exec and command substitution — `uname -m`
  (→ aarch64), a shell computation (sum 1..1000 == 500500), and `head` reading
  the real /proc/version — before powering off (busybox `poweroff` → reboot
  syscall → PSCI SYSTEM_OFF → the emulator halts). The witness asserts these
  markers, and qemu-system-aarch64 -M virt produces them on the same rootfs.

cc37.sha256
  Checksums of the committed artifacts.

Reproduce: ./build.sh  (clones v6.6, builds the kernel + busybox, packs the
layer). The qemu-system-aarch64 differential is run by
vv/suites/cc37-aarch64-devcontainer.sh when qemu is present.
