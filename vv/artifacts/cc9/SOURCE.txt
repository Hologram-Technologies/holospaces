CC-9 — RISC-V emulator ISA-conformance authority artifacts.

The external authority is the RISC-V Unprivileged ISA (RV64I), per
https://riscv.org/technical/specifications/ . Each *.s is a real RV64I program
that computes an ISA-defined result and exits with it (ecall a7=93, a0=code),
in the self-checking style of riscv-tests
(https://github.com/riscv-software-src/riscv-tests). The emulator
(holospaces::emulator) must reproduce the ISA semantics exactly to yield the
expected exit code (expected.txt).

The *.bin are flat binaries (the .text section, loaded at guest address 0 and
the reset PC), assembled from *.s by LLVM's real RISC-V backend — the external
assembler, not hand-encoded:
  Ubuntu LLVM version 18.1.3
  llvm-mc --triple=riscv64 --mattr=+m --filetype=obj <f>.s -o <f>.o
  llvm-objcopy -O binary --only-section=.text <f>.o <f>.bin

Coverage: arithmetic + immediates (sum1to10), logical/shift (bitops), RV64 word
ops + sign extension (wordops), loads/stores LD/SD/LBU (memory).

emulator.wasm
  The system-emulator codemodule: the emulator core (crates/holospaces-emulator,
  wrapping holospaces::emulator) compiled to a hologram Wasm container. It
  imports only `hologram.storage_put` and exports the container ABI
  (hg_init/hg_event/hg_suspend/hg_resume/hg_callback) + memory — so hologram's
  engine runs it as a κ-addressed container (ADR-009; the execution surface,
  CC-6 contract). Rebuild with scripts/build-emulator.sh:
    cd crates/holospaces-emulator
    cargo build --target wasm32-unknown-unknown --release
  rustc 1.96.0 (ac68faa20 2026-05-25)

Zicsr + A extension coverage (assembled with --mattr=+m,+a,+zicsr):
  csr   CSRRW/CSRRWI/CSRRS round-trip through mscratch (587 = 0x123+0x123+5)
  amo   AMOADD.W + AMOSWAP.W on memory (223 = 100 + 123)

tint.S / tint.bin — a CLINT timer-interrupt test (RISC-V Privileged ISA + the
CLINT memory map as the authority), assembled by riscv64-unknown-elf-gcc
(-march=rv64imac_zicsr, -Ttext=0x80000000). Arms mtimecmp via the CLINT, enables
the machine timer interrupt, spins; the emulator raises the timer interrupt
(cause = interrupt|7) into the handler, which verifies mcause and signals pass
over HTIF. tohost in tint.manifest. Witness: tests/cc9_emulator.rs.

sbi.S / sbi.bin — an SBI firmware-interface test (the RISC-V SBI specification as
the authority), assembled by riscv64-unknown-elf-gcc (-march=rv64imac_zicsr).
Drops to supervisor mode via mret, then uses SBI ecalls — console putchar
("OK\n") and system reset — which the emulator-as-SEE (enable_sbi) services.
Witness: tests/cc9_emulator.rs::the_emulator_services_sbi_console_and_shutdown.
