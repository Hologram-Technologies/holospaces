CC-9 — the official RISC-V conformance suite (the canonical ISA authority).

The external authority is riscv-tests — the RISC-V Software project's official
ISA conformance suite, the same suite real hardware and QEMU are validated
against (https://github.com/riscv-software-src/riscv-tests). Each test runs in a
real machine-mode environment (installs mtvec, drops to U-mode via mret, runs
self-checking cases, and signals pass/fail through the HTIF tohost channel).

Build (the *.bin are flat images of the machine-mode '-p' tests, base
0x80000000; manifest.txt pins each test's tohost address since it depends on the
test size):
  FLAGS="-march=rv64g -mabi=lp64 -static -mcmodel=medany -fvisibility=hidden \
         -nostdlib -nostartfiles -I env/p -I isa/macros/scalar -T env/p/link.ld"
  for s in isa/rv64u{i,m,a}/*.S; do
    riscv64-unknown-elf-gcc $FLAGS $s -o <name>
    riscv64-unknown-elf-objcopy -O binary <name> <name>.bin
    # tohost = nm <name> | awk '$3=="tohost"'  -> manifest.txt
  done
