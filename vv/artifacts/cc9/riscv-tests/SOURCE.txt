CC-9 — the official RISC-V conformance suite (the canonical ISA authority).

The external authority is riscv-tests — the RISC-V Software project's official
ISA conformance suite, the same suite real hardware and QEMU are validated
against (https://github.com/riscv-software-src/riscv-tests). Each test runs in a
real machine-mode environment (installs mtvec, drops to U-mode via mret, runs
self-checking cases, and signals pass/fail through the HTIF tohost channel).

Build (the *.bin are flat images of the machine-mode '-p' tests, base
0x80000000; manifest.txt pins each test's tohost address since it depends on the
test size):
  FLAGS="-march=rv64g -mabi=lp64 -static -mcmodel=medany -fvisibility=hidden \
         -nostdlib -nostartfiles -I env/p -I isa/macros/scalar -T env/p/link.ld"
  for s in isa/rv64u{i,m,a}/*.S; do
    riscv64-unknown-elf-gcc $FLAGS $s -o <name>
    riscv64-unknown-elf-objcopy -O binary <name> <name>.bin
    # tohost = nm <name> | awk '$3=="tohost"'  -> manifest.txt
  done

Coverage: rv64ui (base integer), rv64um (multiply/divide), rv64ua (atomics), rv64uc (compressed) —
86 tests (+ the full rv64mi/rv64si privileged suites). Witness: tests/cc9_emulator.rs
(the_emulator_passes_the_official_riscv_tests).

Privileged suites (rv64mi, rv64si) — FULL coverage. The emulator passes ALL of
the machine-mode (rv64mi) and supervisor-mode (rv64si) tests: traps + delegation,
mret/sret, ecall/ebreak, the WARL/read-only CSR semantics (rv64mi/si-csr,
rv64mi-mcsr — mstatus.UXL/SXL fixed at 2, misa read-only), the debug-trigger
absence (rv64mi-breakpoint — a no-trigger hart), misaligned load/store/fetch
(rv64mi/si-ma_*), the FP-disabled trap (mstatus.FS=0 -> illegal), and the
composite vectored-interrupt / S-mode / paging test (rv64mi-illegal — incl. the
mstatus.TVM/TSR trap semantics for sfence.vma/satp/sret). No exclusions.

Floating point (F/D): rv64uf (single) + rv64ud (double) — built with
-mabi=lp64d. The emulator's F/D extension computes correctly-rounded results,
the IEEE-754 accrued exception flags, and all rounding modes on the libm
foundation hologram's float kernels (mathf) are built on (error-free
transformations recover the exact residual for inexact + directed rounding).
All 23 FP tests pass.
